# Definitions and Glossary

| Term | Definition |
| --- | --- |
| Backlog refinement | Ongoing work to clarify, split, prioritize, and prepare backlog items for delivery. |
| Definition of ready | Shared criteria indicating work is sufficiently understood to enter planning. |
| Definition of done | Shared criteria indicating work is complete, verified, and acceptable. |
| Sprint goal | Short statement of the outcome a sprint is intended to achieve. |
| Velocity | Historical amount of work completed by a team, useful for team-level forecasting only. |
| Throughput | Count of work items completed in a period. |
| Cycle time | Time from work start to completion. |
| Lead time | Time from request or commitment to delivery. |
| Carryover | Work planned for a sprint that is not completed in that sprint. |
| Unplanned work | Work introduced outside the planned sprint or delivery commitment. |
| Technical debt | Design, code, test, or operational shortcuts that increase future cost or risk. |
| Finding | Evidence-supported conclusion about a meaningful condition, gap, or risk. |
| Observation | Neutral statement of what was seen, heard, or measured. |
| Risk | Uncertain event or condition that may affect objectives. |
| Recommendation | Proposed action to address a finding or reduce risk. |
| SDLC | Software development life cycle from idea through operation and feedback. |
| AI-SDLC | SDLC activities augmented by AI with governance, traceability, validation, and human accountability. |
| Traceability | Ability to connect evidence, decisions, artifacts, changes, and outcomes. |
| Quality gate | Required check before work advances to the next stage. |
| Human approval gate | Explicit accountable human review before AI-assisted output is accepted or used. |
