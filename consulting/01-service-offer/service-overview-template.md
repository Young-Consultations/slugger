# Service Overview Template

| Field | Working Content |
| --- | --- |
| Service name | Software Delivery and AI-SDLC Alignment Assessment |
| Client problem | Delivery outcomes are unpredictable, prioritization is unclear, or AI-assisted development needs governance and traceability. |
| Ideal client | Software organization with active delivery teams, customer commitments, and leadership support for improvement. |
| Business outcomes | Clear delivery risks, prioritized improvement roadmap, better customer alignment, and measurable delivery governance. |
| Assessment areas | Customer alignment, backlog, prioritization, sprint execution, flow, SDLC maturity, technical debt, communication, metrics, AI-SDLC readiness. |
| Engagement duration | Common range: 2-4 weeks depending on team count, evidence access, and stakeholder availability. |
| Activities | Intake, discovery, evidence review, interviews, scoring, findings analysis, recommendations, readout. |
| Deliverables | Assessment report, scorecard, findings register, recommendations, 30/60/90 roadmap, implementation backlog starter. |
| Client responsibilities | Provide sponsor, stakeholders, approved evidence, tool access where appropriate, and timely review. |
| Assumptions | Assessment is advisory, evidence is complete enough for directional findings, and client owns implementation decisions. |
| Exclusions | Legal advice, security certification, production code changes, personnel performance evaluation, and unmanaged tool access. |
| Optional follow-on services | Fractional delivery leadership, backlog refinement facilitation, sprint operating model improvement, AI-SDLC workflow design. |
| Call to action | Schedule qualification call, confirm scope, and identify evidence owner and executive sponsor. |
