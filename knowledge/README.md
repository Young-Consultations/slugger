# knowledge/

This directory contains the long-term organizational knowledge base for the Slugger project.

## Purpose

Knowledge is a permanent project asset. This directory captures engineering references, lessons learned, reusable patterns, prompt improvements, and best practices that accumulate over time and inform future work.

## Structure

| Path | Contents |
|------|----------|
| `engineering/` | General software engineering references |
| `prompt-engineering/` | Prompt design patterns and improvements |
| `architecture/` | Architecture patterns and examples |
| `python/` | Python development standards and patterns |
| `security/` | Security patterns and OWASP references |
| `testing/` | Testing patterns and strategies |
| `deployment/` | Deployment and infrastructure patterns |
| `design-patterns/` | Object-oriented and functional design patterns |
| `lessons-learned/` | Post-mortems and retrospective findings |
| `decisions/` | Informal decision notes not yet promoted to ADRs |

## Conventions

- Knowledge articles are written in Markdown.
- Articles include the date they were created or last updated.
- Articles are reusable across projects where practical.
- Superseded knowledge is archived, not deleted.

## Related

- `docs/adr/` — formal architectural decisions derived from knowledge
- `prompts/` — prompt templates informed by knowledge articles
- `memory/` — runtime agent memory that may draw from this knowledge base
