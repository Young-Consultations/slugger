# logs/

This directory stores runtime log files generated by the Slugger system.

## Purpose

Logs provide operational visibility into agent activity, workflow execution, errors, and performance. They support debugging, auditing, and continuous improvement.

## Conventions

- Log files are gitignored; they are not committed to version control.
- Log file names include the date and session identifier.
- Log levels follow the standard hierarchy: `DEBUG`, `INFO`, `WARNING`, `ERROR`, `CRITICAL`.
- Structured logging (JSON) is preferred over plain-text logging in production.

## Typical Contents

- `workflow/` — per-workflow execution logs
- `agents/` — per-agent invocation logs
- `errors/` — error and exception logs
- `metrics/` — performance metrics logs

## Note

The `logs/` directory is included in `.gitignore`. Only the directory structure (this README) is tracked.

## Related

- `observability/` — observability infrastructure that writes to this directory
- `metrics/` — metrics collection that complements log data
