# Security Policy

## Supported versions

Security fixes are provided for the current released Slugger package and the active `main` branch. Older releases may receive best-effort guidance, but maintainers may ask reporters to validate against the current supported line before remediation is planned.

## Reporting vulnerabilities privately

Please report suspected vulnerabilities through GitHub private vulnerability reporting for `Young-Consultations/slugger` when available. If private reporting is not available to you, contact the maintainers through a private organization-approved channel and include a minimal, non-sensitive reproduction.

Do not report secrets, tokens, private keys, customer data, or exploit payloads in public issues, pull requests, workflow logs, or discussions.

## Generated-code disclaimer

Slugger-generated projects are draft outputs for human review. Generated examples, especially security or cryptography examples, are educational unless independently reviewed, tested, and approved for production use.

## Expected response workflow

Maintainers triage private reports, validate impact, identify affected versions or workflows, prepare focused fixes, and coordinate disclosure after a safe remediation path exists. Response timing depends on severity, reproducibility, and maintainer availability; this policy does not promise unsupported response-time guarantees.
