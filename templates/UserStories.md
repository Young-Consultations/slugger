# User Stories

Project: [PROJECT_NAME]

How to write stories
- Use the format: "As a <role>, I want <feature> so that <benefit>."
- Keep stories small and testable.
- Link stories to requirement IDs for traceability.

---

## Story template
- ID: US-001
- Title: [Short descriptive title]
- Story: As a [role], I want [capability] so that [benefit].
- Priority: [High/Medium/Low]
- Estimate: [Story points / T-shirt]
- Related requirements: [REQ-xxx]

### Acceptance criteria
- Given [context], when [action], then [expected outcome].
- [Include edge cases and negative scenarios]

### Definition of Done
- Unit tests added and passing
- Integration tests (if applicable) added and passing
- Documentation updated (README / docs)
- Code reviewed and approved

### Notes / Design
- [UI mock links, data schema notes, external API considerations]

### Test cases (examples)
- TC-001: [Automated unit test description]
- TC-002: [Integration or E2E test description]
